# Proceso de construcción

Esta carpeta documenta cómo se construyó Bibl·AI. No es código de producción — es transparencia sobre el proceso.

## Qué hay acá

- **planes-implementacion/**: planes de cada feature antes de implementar (responsive, tablas mobile, etc).
- **sql-historico/**: scripts SQL que se ejecutaron una sola vez (correcciones editoriales, fixes de comillas, datos factuales).
- **bitacora.md**: registro cronológico de decisiones y aprendizajes (si existe).

## Por qué está documentado

Bibl·AI fue construido por una PM no-dev coordinando un equipo de herramientas de IA (Claude, Claude Code, ChatGPT, Midjourney). Documentar el proceso es parte del aprendizaje y permite ver el "detrás de escena" del workflow.
